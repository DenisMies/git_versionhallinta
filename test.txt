Muistiinpanoja git-versionhallinta kurssille

Uusi hakemisto - mkdir (hakemiston nimi)
cd (hakemiston nimi) - vaihetaan hakemistoa
cd .. - hakemistossa mennään yksi kansio taaksepäin
dir - näyttää näyttää hakemistossa olevat kansiot/tiedostot
New-item (tiedoston nimi . muoto) -ItemType File - luo uuden tiedoston esim (test.txt)
ls -a/ls -Force (PowerShell) = näyttää piilotetut tiedostot ls -a ei toiminut powershellissä mahd.kysymys opelle?
code . - avaa koodieditorin

git init - luo tyhjän repositorin hakemistoon
git clone - kopioi olemassa olevan repositoryn
git add (tiedosto) lisää tiedoston tiedot committtia varten
git add . - lisää nykyisen hakemiston tiedot committia varten
git commit -m "" - tallentaa staging area alueella olveat muutokset git repositorioon versionhallintatietona
git push - lähettää paikallisen git repositorion committaukset etärepositorioon (esim githubiin) missä muut näkevät ne
git status - näyttää nykyisen git repositorion tilan (mitkä tiedostot staging areassa, mitkä muuttumattomia ja mitkä ei seurattuja)
git log - näyttää repositorion committien historian
git rm - poistaa tiedoston Git-repositorion seurannasta ja myös yleensä hakemistosta
git mv - siirtää tai uudelleen nimää tiedoston esim git mv oldname.txt newname.txt tai siirtää git mv newname.txt (uuden hakemiston nimi)
